HeadingsMap - Política de privacidad

La presente política de privacidad se aplica a las extensiones del navegador HeadingsMap y WCAG Contrast Checker y tiene por objeto informar a los usuarios sobre los permisos necesarios para su funcionamiento y los motivos por los que son necesarios.

No se recogen ni guardan datos

Las extensiones HeadingsMap y WCAG Contrast Checker no recogen ningún dato de ningún sitio web. Sólo realizan el análisis del HTML en el lado del navegador (lado del cliente) utilizando javascript (sin el uso de bibliotecas o frameworks de terceros) y no recogen, guardan o comparten ningún dato de las páginas web.

Fueron desarrollados para ayudar y apoyar a los desarrolladores, consultores, auditores de accesibilidad y calidad o cualquier otro usuario final en su trabajo diario. No hay intenciones ocultas.

Permisos del navegador

HeadingsMap y WCAG Contrast Checker acceden al HTML de la página que muestra el navegador. Por esta razón, y también porque ofrece al usuario un conjunto de opciones y preferencias que pueden ser guardadas (tema, posición del panel, tipo de análisis, identificación de errores, etc.), requiere algunos permisos específicos del navegador para funcionar.

Ninguno de los permisos que requieren las extensiones recoge datos o contenido web. Se requieren para realizar el análisis, dar al usuario información sobre el html y los estilos que ha aplicado a sus elementos, y ofrecer un conjunto de opciones para definir la forma en que realizar el análisis, así cómo la forma demostrar los resultados. No se recogen datos de los contenidos ni de los sitios en los que el usuario utiliza las extensiones. Además, las extensiones no realizan ninguna acción mientras no están en ejecución.

Esta es la lista de permisos del navegador y las razones por las que son necesarios:

all_urls
Este permiso permite que las extensiones funcionen en cualquier URL
activeTab
El permiso activeTab concede acceso temporal al sitio en el que se encuentra el usuario y permite que la extensión utilice el permiso tabs en la pestaña actual. Este permiso no muestra ninguna advertencia durante la instalación, y se utiliza para:
  • Llamar a tabs.executeScript y tabs.insertCSS en la pestaña actual. La ejecución de script es necesaria para analizar el DOM. La inserción de CSS es necesaria para dar estilo al panel y resaltar elementos (característica que tienen las extensiones que consiste en resaltar los elementos de la página al seleccionar su información en el panel)
  • Identificar la URL y el título de la pestaña a través de una API que devuelve un objeto tabs. La única razón para obtener esta información es la de mostrar al usuario los documentos html que se están viendo y darle la opción de decidir en cuál debe realizar el análisis
storage
Permite guardar (exclusivamente en el navegador) y recuperar las preferencias del usuario, tales como la disposición del panel, el tema, los avisos de error, etc.
webNavigation
Añade "listeners" de eventos para las distintas etapas de una navegación. La navegación consiste en pasar de una URL a otra, normalmente (pero no siempre) en respuesta a una acción del usuario (como hacer clic en un enlace o introducir una URL en la barra de direcciones)
